# # Структура 
Изначально, так как ClickHouse является некой базой данных, была идея использовать StatefulSet и volumeClaimTemplates, но так как в задании четко было указано создать именно single-инстанцию, использовался Deployment.  
Для изучения структуры конфиг-файлов для ClickHouse, была изучена документация на официальном сайте и сторонние статьи, а также проанализированы конфиг-файлы и docker обраще ClickHouse.  
В целях параметризации был использован helm. Таким образом, для изменения версии ClickHouse достаточно в values.yaml поменять поле image.tag, а для ввода пароля для нового пользователя - добавить значение в users, тоже хранящееся в values.yml. Для добавления же самого пользователя, необходимо дополнить по агалогии структуру users, находящуюся в users.xml.
Таким образом ClickHouse, собранный из контейнера, использует 2 persistentVolume: 1 - для хранения данных, 2 - для хранения логов.
